# 📋 如何测试"下一轮"功能

## 🎯 测试步骤

### 步骤 1: 打开个人仪表盘
1. 启动应用程序
2. 导航到 **个人仪表盘** 页面（默认页面）
3. 查看页面中的 **"导师项目参与情况"** 卡片

### 步骤 2: 查看默认轮次
默认情况下，卡片显示：
- 标题：**导师项目参与情况**
- 右上角下拉框显示：**2024年秋季 (当前)**
- 卡片内容：显示您在2024年秋季的导师项目参与情况

### 步骤 3: 切换到"下一轮"
1. 点击右上角的下拉选择器（显示 "2024年秋季 (当前)"）
2. 在下拉菜单中选择 **"下一轮 (即将开始)"**
3. 等待页面刷新

### 步骤 4: 查看琥珀色提示框
选择"下一轮"后，您应该看到：

**琥珀色提示框**（在卡片内容的顶部）：
```

│  ⏰  下一轮 Mentorship 即将开始  [下一轮]        │
│                                                 │
│  📅 开始时间: 2026年3月1日                       │
│  下一轮 Mentorship 尚未开始，                    │
│  请在轮次开始前完成注册信息填写。                │

```

**下方显示空状态**：
```

│          该轮次尚未开始                          │
```

## 🐛 故障排查

### 问题 1: 没有看到琥珀色提示框

**可能原因**：
1. 没有选择"下一轮"，仍然在查看"2024年秋季"
2. 日期设置有问题（但现在已修复为2026年3月1日）
3. 浏览器缓存问题

**解决方法**：
1. 确认下拉选择器显示的是 "下一轮 (即将开始)"
2. 刷新页面（Ctrl+F5 或 Cmd+Shift+R）
3. 清除浏览器缓存后重试

### 问题 2: 日期显示不正确

**检查**：
- 打开 `/test-next-round.html` 查看日期计算是否正确
- 检查浏览器的系统日期是否正确

### 问题 3: 下拉选择器中没有"下一轮"选项

**检查**：
- 确认 `/utils/mockData.ts` 中的 `mentorshipRounds` 包含下一轮数据
- 确认下一轮的 `startDate` 是 '2026-03-01'
